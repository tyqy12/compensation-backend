# 前端动态权限改造开发计划

## 背景与目标

- **问题陈述**：当前前端使用静态菜单与按钮控制，普通用户能看到管理员菜单/入口，缺乏灵活的权限管控。
- **改造目标**：接入后端动态资源树，前端根据用户授权实时生成菜单/路由/按钮权限，实现“所见即所得”。
- **范围**：导航菜单、路由守卫、按钮权限、权限缓存与异常兜底，覆盖工作台、系统管理等核心页面。

## 工作分解（Workstreams）

### 1. 权限模型接入

- 定义前端资源类型 `MenuResource`、`ViewResource`、`ActionResource`。
- 编写 `ResourceService`，封装 `GET /api/auth/me/resources` 与 `GET /api/auth/me/actions`，实现缓存与错误封装。
- 建立 `PermissionContext`（React Context + TanStack Query），提供 `usePermission` 等 Hook。

### 2. 动态导航与路由

- 登录成功后获取资源树，生成动态路由表并注入 React Router。
- Sider 菜单组件根据资源树渲染，支持多级、排序、隐藏。
- 路由守卫：未授权路径重定向 `403`，菜单缺失时隐藏入口。

### 3. 按钮权限控制

- 提供 `usePermission(code)` Hook 与 `PermissionGuard` 组件。
- 改造关键页面按钮：员工详情（标记架构外/指定负责人）、组织同步、集成配置管理等。
- 对应按钮隐藏或禁用时提供 Tooltip 提示（可选）。

### 4. 错误处理与缓存

- 权限接口异常兜底逻辑：重试、静态菜单 fallback、Toast 提示。
- SessionStorage / 内存缓存结合后端版本号，避免重复拉取。
- 权限变更广播（基于版本号）触发刷新。

### 5. 测试与文档

- 单元测试：Mock 权限接口，验证不同角色视图/按钮；
- 集成/E2E：管理员 vs 普通用户导航差异、审批触发场景；
- 文档更新：`docs/routing.md`、`docs/state.md`，新增 `docs/frontend-permissions.md`。

## 关键交付物

- `PermissionContext`、`usePermission` 基础设施。
- 权限驱动的动态菜单与路由。
- 按钮级权限控制覆盖核心管理页面。
- 权限加载兜底体验与提示。
- 完整测试用例与文档。

## 依赖与协作

- **后端**：资源树与动作接口、权限缓存失效机制、冲突审批提示（返回 `workflowId`）。
- **运维**：Redis 缓存配置（按用户/版本维度）；
- **产品/设计**：菜单/按钮与权限 code 映射表、审批提示文案；
- **QA**：管理员/普通用户账号、审批流程测试用例。

## 验收标准

- 未授权用户看不到管理员菜单或按钮，访问返回 `403`；
- 权限变更后刷新或重新登录即可生效；
- 单元测试覆盖核心逻辑，关键路径有集成测试；
- 文档更新，团队成员可按文档扩展或排查权限问题。

## 风险与缓解

| 风险              | 说明           | 缓解措施                                     |
| ----------------- | -------------- | -------------------------------------------- |
| 后端接口延迟上线  | 权限接口未就绪 | 先构建 Mock，接口可用后切换，标记 TODO       |
| 权限树体量大      | 首屏加载耗时   | 缓存、Skeleton、后台刷新                     |
| 老页面权限遗漏    | 范围大，易遗漏 | 制定权限覆盖 checklist，Code Review 强制检查 |
| 审批返回 409 频繁 | 用户体验受影响 | 前端友好提示 `workflowId` 并引导管理员处理   |
