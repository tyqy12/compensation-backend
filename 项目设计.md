## 薪酬助手系统实施计划

---

### 一、项目概述
**目标**：构建支持支付宝优先支付、多平台架构同步、离线员工管理的智能薪酬系统  
**核心能力**：
- ✅ 企业微信/钉钉/飞书三平台架构同步
- ✅ 支付宝批量转账（1000+并发）
- ✅ 离线员工专属管理通道（管理员强制绑定企业微信/钉钉）
- ✅ 动态审批链（架构内+离线双轨）
- ✅ 前端框架无关（React/Vue均可接入）

**合规要求**：
- 支付宝支付符合《非银行支付机构网络支付业务管理办法》
- 身份证信息加密存储（国密SM4+AES-256双保险）
- 操作日志留存≥5年（满足金融审计要求）

### 四、关键配置指南

#### 4.1 支付宝配置要点
```properties
# application-prod.yml
alipay:
  app-id: 2021000000000000  # 企业应用ID
  private-key: |
    -----BEGIN RSA PRIVATE KEY-----
    MIIEowIBAAKCAQEA...
    -----END RSA PRIVATE KEY-----
  public-key: | 
    -----BEGIN PUBLIC KEY-----
    MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQC...
    -----END PUBLIC KEY-----
  notify-url: https://api.your-system.com/alipay/notify  # 异步通知地址
  batch-transfer-limit: 1000000  # 单日转账限额(分)
```
> **必须操作**：  
> 1. 企业支付宝账户开通【单笔转账到支付宝】权限  
> 2. 沙箱环境测试通过后再切换正式环境

#### 4.2 企业微信/钉钉配置
| 平台 | 配置项 | 说明 |
|------|----------|------|
| **企业微信** | • 创建自建应用<br>• 配置可信IP | 1. 获取`corpid`和`secret`<br>2. 开启“审批”权限 |
| **钉钉** | • 创建企业内部应用<br>• 添加审批权限 | 1. 获取`appKey`/`appSecret`<br>2. 配置服务器出口IP白名单 |
| **通用** | `primary_platform` | 管理员必须指定：`ENTERPRISE_WECHAT` 或 `DINGTALK` |

#### 4.3 离线员工管理配置
```yaml
# application.yml
offline:
  manager:
    require-bound-platform: true  # 管理员必须绑定平台
    default-approval-chain: 
      - role: MANAGER  # 管理员自审
      - role: FINANCE # 财务
    sms-fallback: true  # 通知失败自动短信
  payment:
    daily-limit: 500000  # 单人日转账限额(分)
```
